## Quick Hackery

I like being able to debug the code. When I tried to do that with [q4_4.cpp](./ch4/q4_4.cpp), I got this warning:

> ⚠️ warning: GDB: Failed to set controlling terminal: Operation not permitted

And it wouldn't actually run the program or hit breakpoints. It took about an hour of fumbling around, but the conclusion is that I have WSL2 running Ubuntu 26.04, which ships with GDB 17.1.  My VS Code C/C++ extension isn't able to drive that, so I needed to enable the pre-release version of the VS Code C/C++ extension.

After that, everything runs again.
