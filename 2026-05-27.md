## Quick System Hackery

I'm on my Windows machine right now, so I need some tools --
1. Download and install MSYS2, so I have access to UCRT64.
2. From UCRT64 --
```
pacman -S mingw-w64-ucrt-x86_64-gcc mingw-w64-ucrt-x86_64-cmake mingw-w64-ucrt-x86_64-ninja
```
3. Add `C:\msys64\ucrt64\bin` to my system path.

See [./ch1/q1_2.cpp](./ch1/q1_2.cpp). Compile and run with:
```
g++ -std=c++17 -o q1_2.exe q1_2.cpp && q1_2.exe
```

* The current implementation is O(1) space, O(n) time.
* Works for a fixed character set, but not for something like Unicode. In that case, we'd probably want to use a hash map, `std::unordered_map<char, int>`.
* If we switch to a hash map, the space complexity becomes O(k) in the worst case, where k is the size of the character set. Time complexity could become O(k * n), if all hashed to the same bucket. But realistically with a good hash, O(n).
* We could also solve by alphabetizing the strings.
