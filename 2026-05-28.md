## Q1.3

It says, "You may assume that the string has sufficient space at the end to hold the additional characters". So it seems like they want us to modify the string in-place. We could count number of spaces starting at the front... or we could work our way from the back of the string. So I'm going to work my way from the back.

See [./ch1/q1_3.cpp](./ch1/q1_3.cpp). Compile and run with:
```
g++ -std=c++17 -o q1_3.exe q1_3.cpp && q1_3.exe
```

This is O(n) time, O(1) space.

The code would be significantly more readable if we allocated a new `output` string in `urlify()` and copied from `input` to the `output`. That change would make it O(n) time, O(n) space.

## Q1.4

See [./ch1/q1_4.cpp](./ch1/q1_4.cpp). Compile and run with:
```
g++ -std=c++17 -o q1_4.exe q1_4.cpp && q1_4.exe
```

This is O(n) time, O(n) space. (It's O(n) space because of the string allocations in the helper functions.) It could be sped up a few ways:

1. Combine `onlyLetters()` and `toLower()` together. That way it's only one pass. It would still be O(n) though.
2. Actually, just skip `onlyLetters()` and `toLower()` altogether. Just do something like:
```
    if (c >= 'a' && c <= 'z') {
        letter_count[c - 'a']++;
    } else if (c >= 'A' && c <= 'Z') {
        letter_count[c - 'A']++;
    }
```
3. Instead of iterating over `letter_count` to count the number of odds, we could keep track of the number of odds while iterating over `letters`.

If we did suggestions #1 and #2, it would make it O(1) space and much faster.
