## Q2.7

This was straightforward. Two linked lists "intersect" if they share a common node. In such a case, they will also have the same **last** node.

If the linked list has a pointer to its last node, this is O(1) time. Otherwise, there are at least a couple of options:

1. Traverse both lists to find the last node of each. Compare them. See `doListsIntersect()`. This is O(n) time, O(1) space.
2. Traverse one of the lists, and store its nodes into a hashmap. Then traverse the second list, and check if any of its nodes are in the hashmap. I don't think this is actually any faster, since it involves inserting every node into the hashmap.

**Maybe** it would be faster to zipper-traverse the two different lists, and check if the current node is in the hashmap. If so, return true. If not, add the node to the hashmap.

This would work well for extremely long linked lists that **do** intersect near the start. But it is always worse if the two lists **do not** intersect.

```
->1->2->3->4->5->6->7->8->9
        ^                 |
        +-----------------+
  x  y
     x     y
        x        y
           x           y
        y     x
              y  x
                    xy
```

So we see that the two pointers eventually meet. And if the beginning of the loop is later in the linked list, the two pointers will meet earlier in the loop. (There will be more steps until they reach the start of the loop again.)

`k` = The first node of the loop
`n` = The length of the linked list
`L` = The length of the loop.

If the "slow" pointer moves `k + m` steps, then the "fast" one moves `2 * (k + m)` steps. So there's a `(k + m)` difference. And we know that `(k + m)` differs by some multiple of the loop length. So:

`k + m = p*L`
`k = p*L - m`

Again, the "slow" pointer is at `k + m`. It is `m` steps past the loop start. If it travels another `k` steps from there, it is `k + m` steps **past the loop start**. And we know that `k + m = p*L`, so then the "slow" pointer would have travelled `p` full loops, So it would be at the loop start.

If we start another pointer at the beginning of the linked list, then it and the "slower" pointer will reach the start of the cycle after `k` steps.

**Note**: I asked Claude Code to add example tests, and it referred to this algorithm as "standard Floyd's" (a.k.a. Floyd's Cycle Detection Algorithm). That was tough to figure out, but it's creative.
