## Q3.6

This was fun. I initially wrote the `AnimalShelter` class with member variables:

```
ctci::jim::Queue<Animal> animals;
ctci::jim::Queue<Animal> skippedAnimals;
```

That doesn't work at all, because then we can't downcast from an `Animal` object to a `Cat` object or `Dog` object. If it's `Queue<Animal>`, then any properties of the child class objects are lost when they are copied into the queue. The queue only allocates `sizeof(Animal)` bytes for each element of the queue.

So we have to switch to pointers (`Animal*`)... or possibly use references, but that gets messy with ensuring the referenced objects are never destroyed while they are in the queue.

**Note**: My approach with `skippedAnimals` won't work with three or more animal types. We would need to do something more complex then.
